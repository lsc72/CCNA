Création de 2 sous-réseaux

Pour comprendre comment appliquer un sous-réseau /25 dans un réseau, observez la topologie de la figure 1. Deux segments de réseau local sont reliés aux interfaces GigabitEthernet du routeur R1. L'un des sous-réseaux est attribué à chacun des LAN.

La figure 2 indique les adresses importantes du premier sous-réseau, 192.168.1.0/25. Notez ce qui suit :

  • L'adresse réseau IPv4 est 192.168.1.0 et contient tous les bits 0 de la partie hôte de l'adresse.

  • La première adresse IPv4 d'hôte est 192.168.1.1 et contient tous les bits 0 plus un bit 1 tout à droite de la partie hôte de l'adresse.

  • La dernière adresse IPv4 d'hôte est 192.168.1.126 et contient tous les bits 1 plus un bit 0 tout à droite de la partie hôte de l'adresse.

  • L'adresse de diffusion IPv4 est 192.168.1.127 et contient tous les bits 1 de la partie hôte de l'adresse.

La figure 3 indique les adresses importantes du deuxième sous-réseau, 192.168.1.128/25.

Une adresse IP appartenant à la plage d'hôtes appropriée pour le sous-réseau attribué doit être configurée sur les interfaces du routeur. Les hôtes de ce réseau utiliseront cette adresse comme passerelle par défaut. Il est très courant d'attribuer la première ou la dernière adresse disponible d'une plage réseau aux interfaces du routeur. La figure 4 illustre la configuration des interfaces de R1 avec la première adresse IP pour leurs sous-réseaux respectifs à l'aide de la commande de configuration d'interface ip adress.

Une adresse IPv4 et une passerelle par défaut doivent être configurées sur les hôtes de chaque sous-réseau. La figure 5 indique la configuration IPv4 de l'hôte PC2 sur le réseau 192.168.1.128/25. Notez que l'adresse IPv4 de la passerelle par défaut est l'adresse configurée sur l'interface G0/1 de R1, 192.168.1.129, et que le masque de sous-réseau est 255.255.255.128.

Figure 1 shows a topology with one router connected to two switches to create two LANs. The subnets 192.168.1.0/25 and 192.168.1.128/25 are assigned to the two LANs. Figure 2 shows the network address, first host address, last host address, and broadcast address for the 192.168.1.0/25 subnet. The last octet is in binary to show that a network address is all zeros in the host portion. The first host address is all zeros except the least significant bit in the host portion. The last host address is all ones except for the least significant bit in the host portion. And the broadcast address is all ones in the host portion. Similar to Figure 2, Figure 3 shows the network address, first host address, last host address, and broadcast address for the 192.168.1.128/25 subnet with the last octet shown in binary.
