Configurer les mots de passe

Le mot de passe le plus important à configurer est celui permettant d'accéder au mode d'exécution privilégié, comme illustré à la figure 1. Pour sécuriser l'accès au mode d'exécution privilégié, utilisez la commande de config. globale enable secret mot de passe.

Pour sécuriser l'accès au mode d'exécution utilisateur, le port de console doit être configuré, comme illustré à la figure 2. Passez en mode de configuration de console de ligne à l'aide de la commande de configuration globale line console 0. Le zéro sert à représenter la première (et le plus souvent, la seule) interface de console. Spécifiez ensuite le mot de passe du mode d'exécution utilisateur à l'aide de la commande de mot de passe password mot de passe. Enfin, activez l'accès d'exécution utilisateur à l'aide de la commande login. La console d'accès requiert à présent un mot de passe avant d'accéder au mode d'exécution utilisateur.

Les lignes VTY (terminal virtuel) activent l'accès à distance au périphérique. Pour sécuriser les lignes VTY utilisées pour SSH et Telnet, passez en mode ligne VTY à l'aide de la commande de config. globale line vty 0 15, comme illustré à la figure 3. De nombreux commutateurs Cisco prennent en charge jusqu'à 16 lignes VTY, numérotées de 0 à 15. Spécifiez ensuite le mot de passe VTY à l'aide de la commande password. En dernier lieu, activez l'accès VTY à l'aide de la commande login.

Figure 1 demonstrates how to configure an encrypted privileged exec mode password using the "enable secret" command. Figure 2 demonstrates how to configure a password for the console line. Figure 3 demonstrates how to configure a password for the vty lines. Both console and vty lines are configured using the “password” and “login” commands.
