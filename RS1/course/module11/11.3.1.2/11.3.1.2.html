Extensions de la commande ping

Cisco IOS propose un mode « étendu » de la commande ping. Pour entrer dans ce mode, saisissez ping en mode d'exécution privilégié sans spécifier l'adresse IP de destination. Comme le montre la figure, une série d'invites apparait : Il suffit d'appuyer sur Entrée pour accepter les valeurs par défaut indiquées. L'exemple montre comment forcer une requête ping à utiliser 10.1.1.1 comme adresse source (voir R2 sur la figure). L'adresse source d'une requête ping standard serait 209.165.200.226. L'administrateur réseau peut ainsi vérifier à partir de R2 que R1 dispose d'une route vers 10.1.1.0/24.

Remarque : la commande ping ipv6 est utilisée pour les ping IPv6 étendus.
