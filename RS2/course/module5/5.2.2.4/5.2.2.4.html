Sécurité des ports : configuration

La Figure 1 récapitule les paramètres par défaut de la sécurité des ports sur un commutateur Cisco Catalyst.

La Figure 2 illustre les commandes de l'interface de ligne de commande du logiciel Cisco IOS nécessaires pour configurer la sécurité des ports sur le port Fast Ethernet F0/18 du commutateur S1. Remarquez que l’exemple ne précise aucun mode de violation. Dans cet exemple, le mode de violation est Shutdown (mode par défaut).

La Figure 3 montre comment activer la sécurité des ports à l'aide des adresses MAC sécurisées rémanentes sur le port Fast Ethernet 0/19 du commutateur S1. Comme indiqué précédemment, le nombre d'adresses MAC sécurisées peut être configuré manuellement. Dans cet exemple, la syntaxe des commandes Cisco IOS est utilisée pour définir le nombre d'adresses MAC maximal sur 10 pour le port 0/19. Le mode de violation est défini sur Shutdown, le mode par défaut.

La Figure 1 résume la configuration par défaut de la sécurité des ports sur un commutateur Cisco Catalyst. La sécurité des ports est désactivée lorsque le nombre maximal d’adresses MAC sécurisées défini sur la valeur 1, le mode de violation est arrêté et l’apprentissage d’adresse MAC rémanente est désactivé. La Figure 2 affiche les commandes de l’interface de ligne qui sont nécessaires pour configurer la sécurité des ports sur une interface de port de commutateur. La première commande correspond à « interface fast Ethernet 0/18 ». Cette commande passe du mode de configuration globale à une interface spécifique. La commande suivante est « switchport mode access » qui définit le port comme un port d’accès strict pour un périphérique final. La dernière commande est « switchport port-security ». La Figure 3 continue l’exemple précédent et ajoute la commande « switchport port-security maximum 50 ». La dernière commande est « switchport port-security mac-address sticky ».
