La Figure 1 présente un graphique pour les routes correspondantes. Dans la figure, si la route finale de niveau 1 est la meilleure correspondance, alors le paquet est acheminé en suivant l’adresse IP du tronçon suivant ou en suivant l’interface de sortie. La Figure 2 illustre le processus si la meilleure correspondance pour le paquet est une route parent de niveau 1.  Si une correspondance avec une route enfant de niveau 2 est trouvée, alors le paquet est acheminé en suivant l’adresse IP du tronçon suivant ou en suivant l’interface de sortie.  La Figure 3 illustre le processus si aucune correspondance avec une route enfant de niveau 2 n’est trouvée, ce qui consiste à rechercher des routes de super-réseau, notamment la route par défaut dans la table de routage.  Si aucune correspondance n’est trouvée, alors le paquet est ignoré.
