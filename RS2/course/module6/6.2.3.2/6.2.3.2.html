VLAN manquants

S’il n’y a toujours aucune connexion entre les périphériques d’un VLAN, mais que les problèmes d’adressage IP ont été réglés, consultez le diagramme de la Figure 1 pour le dépannage :

Étape 1. Utilisez la commande show vlan pour vérifier que le port appartient au VLAN prévu. Si le port est attribué au mauvais VLAN, utilisez la commande switchport access vlan pour corriger l’appartenance. Utilisez la commande show mac address-table pour vérifier que les adresses ont été apprises sur un port spécifique du commutateur et le VLAN auquel le port est attribué, comme illustré dans la Figure 2.

Étape 2. Si le VLAN auquel est affecté le port est supprimé, le port devient inactif. Les ports d’un VLAN supprimé ne seront pas répertoriés dans la sortie de la commande show vlan. Utilisez la commande show interfaces switchport pour vérifier si le VLAN inactif est attribué au port, comme illustré dans la Figure 2.

L’exemple de la Figure 2 affiche les adresses MAC qui ont été acquises sur l’interface F0/1. Vous constatez que l’adresse MAC 000c.296a.a21c a été acquise sur l’interface F0/1 du VLAN 10. Si ce chiffre n’est pas le numéro VLAN voulu, modifiez l’appartenance des ports à l’aide de la commande switchport access vlan.

Chaque port d’un commutateur appartient à un VLAN. Si le VLAN auquel appartient le port est supprimé, le port devient inactif. Tous les ports appartenant au VLAN qui a été supprimé ne peuvent plus communiquer avec le reste du réseau. Utilisez la commande show interface f0/1 switchport pour vérifier si le port est inactif. Si le port est inactif, il n’est pas opérationnel tant que le VLAN manquant n’est pas créé à l’aide de la commande de configuration globale vlan vlan-id ou tant que le VLAN n’est pas supprimé du port avec la commande no switchport access vlan vlan-id .

La Figure 1 est un diagramme qui illustre les étapes à suivre pour diagnostiquer les problèmes de connectivité lorsque les VLAN sont en service. La Figure 2 montre la sortie de la commande « show mac address-table interface fast ethernet 0/1 » sur le commutateur 1 et la sortie de la commande « show interfaces fast ethernet 0/1 switchport » sur le commutateur 1.
