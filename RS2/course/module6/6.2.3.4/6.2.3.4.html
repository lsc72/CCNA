Problèmes courants avec les agrégations

Les problèmes de trunking sont généralement associés à des configurations incorrectes. Lors de la configuration de VLAN et de trunks sur une infrastructure commutée, les types d’erreurs de configuration les plus courants sont les suivants :

  • Non-concordance de VLAN natif : les ports trunk sont configurés avec différents VLAN natifs. Cette erreur de configuration génère des notifications de console et peut entraîner, entre autres, des problèmes de routage inter-VLAN. Cela représente un risque de sécurité.

  • Non-concordance du mode trunk : l’un des ports trunk est configuré dans un mode qui n’est pas compatible avec le trunking du port correspondant. En présence de cette erreur de configuration, la liaison agrégée cesse de fonctionner. Assurez-vous que les deux côtés du trunk sont configurés avec la commande switchport mode trunk. Ce cours n’a pas pour objet les commandes de configuration de trunk.

  • VLAN autorisés sur les trunks : la liste des VLAN autorisés sur un trunk n’a pas été mise à jour pour tenir compte des critères de trunking de VLAN actuels. Dans ce cas, un trafic imprévu, voire aucun trafic n’est transmis sur le trunk.

Si un problème est détecté sur un trunk et que la cause en est inconnue, commencez par rechercher une éventuelle non-concordance de VLAN natif sur les liaisons. Si ce n’est pas l’origine du problème, vérifiez s’il ne s’agit pas d’une non-concordance du mode trunk et contrôlez enfin la liste des VLAN autorisés sur le trunk.
