La Figure 1 présente la commande utilisée pour configurer un nom d'hôte sur un routeur.  Les commandes commencent à partir du mode d'exécution utilisateur.  La première commande est « configure terminal » pour ensuite passer au mode de configuration globale.  La commande suivante est « hostname R1 ».  Le nom d'hôte du routeur est maintenant configuré. La Figure 2 configure le routeur pour exiger des mots de passe pour l'accès.  Les commandes commencent en mode de configuration globale.  La première commande définit un mot de passe sur le routeur en mode d'autorisation d'accès ; il s'agit de « enable secret class ».  Le mot clé secret dit au routeur de chiffrer le mot de passe dans le fichier de configuration, et le mot de passe utilisé est « class ».  Ensuite le routeur passe au mode de configuration de console avec la commande « line console 0 ».  Une fois en mode de configuration de console, la commande pour définir un mot de passe est « password cisco » et « cisco » est le mot de passe.  La commande suivante indique au routeur que le mot de passe est requis pour la connexion et que la commande utilisée est « login ».  Ensuite le routeur passe au mode de configuration de lignes de terminal avec la commande de ligne « v.t.y. 0 4 ».  L’accès Telnet sur cinq connexions sera par la suite restreint.  Une fois en mode de configuration de lignes de terminal, la commande pour définir un mot de passe est « password cisco » et « cisco » est le mot de passe.  La commande suivante indique au routeur que le mot de passe est requis pour la connexion et que la commande utilisée est « login ».  La commande suivante demande au routeur de chiffrer tous les mots de passe dans le fichier de configuration.  La commande est exécutée en mode de configuration globale ; il s'agit de « service password-encryption ». La Figure 3 configure un message de bannière que le routeur affiche lors de la connexion d’un utilisateur. La commande s’exécute en mode de configuration globale ; il s’agit de « banner motd $AUTHORIZED ACCESS ONLY$ ».  Le symbole du dollar ($) est utilisé pour indiquer au routeur le début et la fin du message de bannière.  Le symbole peut être n'importe quel symbole, mais il ne peut pas être dans le message. La Figure 4 illustre la commande utilisée pour enregistrer le fichier de configuration de la mémoire vive vers la mémoire vive non volatile.  La commande est exécutée en mode d'exécution privilégié ; il s'agit de « copy running-config startup-config ». La Figure 5 est un exercice interactif qui permet à l'étudiant de configurer un nom d'hôte, de limiter l'accès grâce à des mots de passe et de configurer une bannière sur un routeur.
