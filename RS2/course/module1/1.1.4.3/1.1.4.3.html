Filtrage des résultats de commande show

Les commandes qui génèrent plusieurs écrans de sortie sont, par défaut, mises en pause après 24 lignes. À la fin de cette interruption, le texte « --More-- » s'affiche. Appuyez sur Enter pour afficher la ligne suivante et appuyez sur la touche Espace pour afficher la série de lignes suivante. Utilisez la commande terminal length pour indiquer le nombre de lignes à afficher. La valeur 0 (zéro) empêche le routeur de s'arrêter entre les écrans de résultat.

Une autre fonctionnalité très utile pour améliorer l'expérience utilisateur dans l'interface en ligne de commande est le filtrage obtenu par la commande show. Les commandes de filtrage permettent d'afficher des sections de résultat spécifiques. Pour activer la commande de filtrage, tapez le symbole | après la commande show, puis saisissez un paramètre de filtrage et une expression de filtrage.

Après ce symbole, les paramètres de filtrage suivants peuvent être configurés :

  • section : affiche l'intégralité de la section commençant par l'expression de filtrage

  • include : inclut toutes les lignes de résultat correspondant à l'expression de filtrage

  • exclude : exclut toutes les lignes de résultat correspondant à l'expression de filtrage

  • begin : affiche toutes les lignes de résultat à partir d'un certain point, en commençant par la ligne qui correspond à l'expression de filtrage

Remarque : les filtres de résultat peuvent être utilisés en combinaison avec toute commande show.

Utilisez le contrôleur de syntaxe de la Figure 5 pour filtrer les résultats.

La Figure 1 affiche un exemple d’utilisation des commandes de filtrage sur IOS Cisco. Dans l'exemple, la commande « show running-config pipe section line vty » est saisie et le résultat correspond à la section de la configuration en cours pour les lignes vty. La Figure 2 illustre un exemple d’utilisation d’une commande de filtrage ; il s’agit de « show ip interface brief pipe include up ». La sortie répertorie trois interfaces qui sont opérationnelles, c.-à-d. actives. La Figure 3 illustre un exemple d’utilisation d’une commande de filtrage ; il s’agit de « show ip interface brief pipe exclude unassigned ». La sortie répertorie trois interfaces qui sont opérationnelles, c.-à-d. actives. La quatrième interface du routeur n'est visiblement pas répertoriée dans la sortie étant donné qu'aucune adresse ip ne lui a été attribuée. La Figure 4 illustre un exemple d’utilisation d’une commande de filtrage ; il s’agit de « show ip interface route pipe begin Gateway ». Le résultat répertorie les contenus de la commande « show ip route » qui commence par le mot clé « gateway », jusqu'à la fin de la sortie. La Figure 5 est un élément multimédia interactif qui permet à l’étudiant de s’entraîner à utiliser les commandes de filtrage.
