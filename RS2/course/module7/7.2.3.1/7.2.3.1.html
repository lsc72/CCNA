La commande access-class

Vous pouvez améliorer la sécurité des lignes administratives en limitant l'accès VTY. La restriction de l'accès VTY est une technique vous permettant de définir les adresses IP avec un accès à distance au processus EXEC du routeur. Vous pouvez indiquer les adresses IP autorisées à avoir l’accès à distance à votre routeur via une liste de contrôle d'accès et l'instruction access-class configurés sur vos lignes VTY. Utilisez cette technique avec SSH pour renforcer la sécurité de tout accès administratif.

La commande access-class configurée en mode de configuration de ligne limite les connexions entrantes et sortantes entre un VTY spécifique (vers un périphérique Cisco) et les adresses renseignées dans une liste de contrôle d'accès.

La syntaxe de la commande access-class est la suivante :

Router(config-line)# access-class access-list-number { in [ vrf-also ] | out }

Le paramètre in limite les connexions entrantes entre les adresses de la liste d'accès et le périphérique Cisco, tandis que le paramètre out limite les connexions sortantes entre un périphérique Cisco spécifique et les adresses de la liste d'accès.

La Figure 1 présente un exemple où une plage d'adresses est autorisée à accéder aux lignes VTY 0 à 4. La liste de contrôle d'accès de la figure est configurée pour autoriser le réseau 192.168.10.0 à accéder aux lignes VTY 0 à 4, mais refuser tous les autres réseaux.

Vous devez prendre en compte les éléments suivants lors de la configuration de listes de contrôle d’accès sur des lignes VTY :

  • Vous pouvez appliquer aux lignes VTY aussi bien les listes d'accès nommées que celles numérotées.

  • Vous devez définir les mêmes restrictions sur toutes les lignes VTY car un utilisateur peut tenter de se connecter à n’importe laquelle.

Utilisez le contrôleur de syntaxe de la Figure 2 pour vous entraîner à sécuriser les accès VTY.

Remarque : les listes d’accès s'appliquent aux paquets traversant un routeur. Elles ne sont pas destinées à bloquer les paquets créés sur le routeur. Par défaut, une liste de contrôle d'accès sortante n'empêche pas les connexions d'accès à distance lancées par le routeur.

La Figure 1 affiche un routeur avec un segment de réseau local sur chacune des deux interfaces Gigabit Ethernet. Chaque segment se compose d'un commutateur et d'un ordinateur. Le routeur dispose des ports VTY 0 à 4 configurés pour l'accès. La figure illustre les commandes requises pour créer une liste de contrôle d'accès qui limite les connexions VTY. La figure débute en mode de configuration globale puis se déplace aux lignes VTY avec la commande « line vty 0 4 ». La commande suivante configure le routeur pour utiliser une base de données locale pour l'accès avec la commande « login local ». La ligne suivante définit le type de connexion à autoriser. La commande est « transport input ssh ». Ensuite intervient l'application d'une liste de contrôle d'accès sur les lignes avec la commande « access-class 21 in ». La figure passe du mode de configuration de ligne au mode de configuration globale avec la commande « exit ». La liste de contrôle d'accès est créée avec la commande « access-list 21 permit 192.168.10.0 0.0.0.255 » et la commande « access-list 21 deny any ». La Figure 2 est un élément multimédia interactif qui permet à l’étudiant de configurer les lignes VTY pour accepter SSH sur un routeur.
