Configuration et vérification du protocole NTP

Tant que le protocole NTP n'est pas configuré sur le réseau, la commande show clock affiche l'heure actuelle sur l'horloge logicielle. L'option detail permet également d'afficher la source temporelle. Comme l'illustre la Figure 1, l'horloge logicielle a été configurée manuellement. Utilisez la commande NTP server ip-address en mode de configuration globale pour configurer 209.165.200.225 en tant que serveur NTP pour R1. Pour vérifier si la source temporelle est définie sur NTP, utilisez à nouveau la commande show clock detail.

Comme l'illustre la Figure 2, les commandes show ip ntp associations et show ntp status vérifient que R1 est synchronisé avec le serveur NTP sur 209.165.200.225. Notez que R1 est synchronisé avec un serveur NTP de strate 1 à l'adresse 209.165.200.225, qui est synchronisée avec une horloge GPS. La commande show ntp status permet de voir que R1 est désormais un périphérique de strate 2 synchronisé avec le serveur NTP 209.165.220.225.

L'horloge sur S1 est configurée pour se synchroniser sur R1, comme illustré à la Figure 3. Le résultat de la commande show ntp associations vérifie que l'horloge sur S1 est désormais synchronisée avec R1 sur 192.168.1.1 via NTP. R1 est un périphérique de strate 2 et un serveur NTP sur S1. S1 est désormais un périphérique de strate 3 capable de fournir un service NTP à d'autres périphériques du réseau, tels que des périphériques finaux.
