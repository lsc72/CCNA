Vérification d'une route statique par défaut

Dans la figure, la commande show ip route static affiche le contenu des routes statiques dans la table de routage. Notez l’astérisque (*) en regard de la route avec le code « S ». Comme affiché dans la table des codes de la figure, l'astérisque indique que cette route statique est la route par défaut idéale, c'est pourquoi elle est sélectionnée comme passerelle de dernier recours.

L'élément essentiel de cette configuration est le masque /0. Le masque de sous-réseau dans une table de routage détermine le nombre de bits devant correspondre entre l’adresse IP de destination du paquet et la route dans la table de routage. Le chiffre binaire 1 indique que les bits doivent correspondre. Le chiffre binaire 0 indique que les bits ne doivent pas correspondre. Le masque /0 dans cette entrée de route indique qu'aucun des bits ne doit correspondre. La route statique par défaut correspond à tous les paquets pour lesquels il n'existe pas de correspondance plus spécifique.

La figure affiche le schéma standard de l’atelier avec la sortie de la commande « show ip route static ». Elle indique les relations entre la passerelle de dernier recours au sommet de la sortie et la route statique par défaut en bas.
