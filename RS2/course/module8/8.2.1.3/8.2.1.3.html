SLAAC et DHCPv6

Ce sont les paramètres indiqués dans le message RA qui déterminent si un client est configuré pour obtenir automatiquement ses informations d'adresse IPv6 via SLAAC, DHCPv6 ou une combinaison des deux.

Il s'agit de l'indicateur de configuration des adresses gérée (indicateur M) et de l'indicateur de configuration autre (indicateur O).

Différentes combinaisons des indicateurs M et O permettent aux messages RA d'indiquer une des trois options d'adressage pour le périphérique IPv6, comme illustré sur la figure :

  • SLAAC (annonce de routeur uniquement)

  • DHCPv6 sans état (annonce de routeur et DHCPv6)

  • DHCPv6 avec état (DHCPv6 uniquement)

Quelle que soit l'option utilisée, le document RFC 4861 recommande que tous les périphériques IPv6 effectuent une détection des adresses en double (DAD) sur toutes les adresses de monodiffusion, y compris les adresses configurées via SLAAC ou DHCPv6. La DAD est mise en œuvre à l'aide de l'ICMPv6, spécifié par le document RFC 4443.

Remarque : bien que le message RA indique le processus à utiliser par le client pour obtenir une adresse IPv6 dynamique, le système d'exploitation du client peut choisir d'ignorer le message RA et d'utiliser exclusivement les services d'un serveur DHCPv6.

La figure présente les trois options de message d'annonce de routeur. Le premier est SLAAC, dans lequel le routeur demande à l'ordinateur d'utiliser uniquement les informations du message d'annonce de routeur. Il s'agit du paramètre par défaut. La deuxième option est DHCPv6 sans état qui utilise à la fois SLAAC et DHCP. Dans cette option, le routeur fournit des informations dans un message d'annonce de routeur, mais demande également à l'ordinateur d'obtenir des données auprès d'un serveur DHCPv6. L'option trois est DHCP avec état. Dans cette option, le routeur demande à l'ordinateur d'obtenir toutes les informations d'un serveur DHCPv6.
