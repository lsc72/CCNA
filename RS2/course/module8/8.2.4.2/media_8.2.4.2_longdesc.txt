La Figure 1 présente les commandes utilisées pour configurer les services DHCPv6 avec état sur un routeur.  L’exemple commence en mode de configuration globale et les commandes sont comme suit.  La première commande est : i.p.v.6. unicast-routing.  La commande suivante est : d.h.c.p. pool i.p.v.6.  -stateful.  La commande suivante est : address prefix 2001:db8:cafe:1::/64 lifetime infinite.  La commande suivante est : d.n.s.-server 2001:db8:cafe:aaaa::5.  La commande suivante est : domain-name example.com.  Le pool DHCPv6 a été configuré et le jeu suivant de commandes associe le pool à l’interface.  La CLI dans l’exemple passe d’abord à l’interface locale à l’aide de la commande : exit. La commande exit est suivie par la commande : interface g 0/1.  La commande suivante est : i.p.v.6. address 2001:db8:cafe:1::1/64.  La commande suivante est : d.h.c.p. pool i.p.v.6.  -stateful.  La dernière commande est : i.p.v.6. n.d. managed-config-flag.  La commande « address prefix » et la commande « ipv6 nd managed » sont mises en surbrillance pour indiquer que ces commandes sont spécifiques à la configuration avec état.  La figure répertorie également les commandes pour configurer les services DHCPv6 sans état.  L’exemple commence en mode de configuration globale et les commandes sont comme suit.  La première commande est : i.p.v.6. unicast-routing.  La commande suivante est : d.h.c.p. pool i.p.v.6.-stateless.  La commande suivante est : d.n.s.-server 2001:db8:cafe:aaaa::5.  La commande suivante est : domain-name example.com.  Le pool DHCPv6 a été configuré et le jeu suivant de commandes associe le pool à l’interface.  La CLI dans l’exemple passe d’abord à l’interface locale à l’aide de la commande : exit. La commande exit est suivie par la commande : interface g 0/1.  La commande suivante est : i.p.v.6. address 2001:db8:cafe:1::1/64.  La commande suivante est : d.h.c.p. pool i.p.v.6.-stateless.  La dernière commande est : i.p.v.6. n.d. other-config-flag.  La commande « ipv6 nd other-config-flag » est mise en surbrillance dans la figure pour la désigner comme étant spécifique à la configuration DHCPv6 sans état. La Figure 2 montre le résultat pour trois configurations différentes en exécutant la commande « show ipv6 interface » sur une interface de routeur spécifique.  La première sortie est pour SLAAC et la figure met en surbrillance la ligne qui déclare « hosts use stateless autoconfig for address » (les hôtes utilisent l'autoconfiguration sans état pour l'adresse).  La sortie suivante concerne DHCPv6 sans état et la figure met en surbrillance la ligne qui déclare « hosts use d.h.c.p. to obtain other configuration » (les hôtes utilisent DHCP pour obtenir une autre configuration).  La troisième sortie concerne DHCPv6 avec état et la figure met en surbrillance la ligne qui déclare « hosts use d.h.c.p. to obtain routable addresses » (les hôtes utilisent DHCP pour obtenir des adresses routables).
