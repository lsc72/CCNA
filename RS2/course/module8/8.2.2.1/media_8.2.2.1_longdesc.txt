La Figure 1 présente les commandes utilisées pour configurer DHCPv6 sans état sur un routeur.  La figure débute en mode de configuration globale et la première commande est « IPv6 unicast-routing ».  Cette commande active le routage IPv6.  La commande suivante est « IPv6 DHCP pool » et le nom du pool suit.  Un serveur DNS est configuré avec la commande « dns-server ». Le nom de domaine est configuré avec la commande « domain-name ».  Une interface est configurée avec le pool DHCP avec la commande « IPv6 DHCP server » suivie du nom de pool.  La dernière commande est : i.p.v.6. n.d. other-config-flag. La Figure 2 illustre un exemple complet de configuration d'un routeur en tant que serveur DHCPv6 sans état.  La figure commence en mode de configuration globale et les commandes sont comme suit.  La première commande est : i.p.v.6. unicast-routing.  La commande suivante est : d.h.c.p. pool i.p.v.6.-stateless.  La commande suivante est : d.n.s.-server 2001:db8:cafe:aaaa::5.  La commande suivante est : domain-name example.com.  Le pool DHCPv6 a été configuré et le jeu suivant de commandes associe le pool à l’interface.  La CLI dans l’exemple passe d’abord à l’interface locale à l’aide de la commande : exit. La commande exit est suivie par la commande : interface g 0/1.  La commande suivante est : i.p.v.6. address 2001:db8:cafe:1::1/64.  La commande suivante est : d.h.c.p. pool i.p.v.6.  -stateless.  La dernière commande est : i.p.v.6. n.d. other-config-flag.
