Configuration d'un routeur en tant que serveur DHCPv6 sans état

Comme l'illustre la Figure 1, quatre étapes sont nécessaires à la configuration d'un routeur en tant que serveur DHCPv6 :

Étape 1. Activation du routage IPv6

La commande ipv6 unicast-routing doit être exécutée pour activer le routage IPv6. Cette commande n'est pas nécessaire pour que le routeur agisse comme un serveur DHCPv6 sans état, mais elle est requise pour qu'il récupère des messages RA ICMPv6.

Étape 2. Configuration d'un pool DHCPv6

La commande ipv6 dhcp pool pool-name crée un pool et entre dans le routeur en mode de configuration DHCPv6, identifié par l’invite Router(config-dhcpv6)#.

Étape 3. Configuration des paramètres du pool

Lors du processus SLAAC, le client a reçu les informations dont il avait besoin pour créer une adresse de monodiffusion globale IPv6. Le client a également obtenu les informations sur la passerelle par défaut à l'aide de l'adresse IPv6 source du message RA, qui est l'adresse link-local du routeur. Cependant, le serveur DHCPv6 sans état peut être configuré pour fournir d'autres informations qui n'ont pas forcément été incluses dans le message RA, notamment l'adresse du serveur DNS et le nom de domaine.

Étape 4. Configuration de l'interface DHCPv6

La commande ipv6 dhcp server pool-name en mode de configuration d'interface relie le pool DHCPv6 à l'interface. Le routeur répond aux requêtes DHCPv6 sans état sur cette interface avec les informations contenues dans le pool. L'indicateur O doit être modifié pour passer de 0 à 1 à l'aide de la commande d'interface ipv6 nd other-config-flag. Les messages RA envoyés sur cette interface indiquent que des informations supplémentaires sont disponibles auprès d'un serveur DHCPv6 sans état.

Exemple de serveur DHCPv6 sans état

La Figure 1 présente les commandes utilisées pour configurer DHCPv6 sans état sur un routeur. La figure débute en mode de configuration globale et la première commande est « IPv6 unicast-routing ». Cette commande active le routage IPv6. La commande suivante est « IPv6 DHCP pool » et le nom du pool suit. Un serveur DNS est configuré avec la commande « dns-server ». Le nom de domaine est configuré avec la commande « domain-name ». Une interface est configurée avec le pool DHCP avec la commande « IPv6 DHCP server » suivie du nom de pool.
