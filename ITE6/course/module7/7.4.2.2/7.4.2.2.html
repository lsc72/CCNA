Fonctionnalités de la couche transport

Les données transitant entre une application source et une application de destination sont appelées « conversation ». Un ordinateur peut gérer plusieurs conversations entre plusieurs applications en même temps, comme illustré ci-contre. Ceci est rendu possible par les trois fonctionnalités principales de la couche transport :

  • Suivi de chaque conversation entre les applications - Sur un périphérique, plusieurs applications peuvent utiliser le réseau en même temps.

  • Segmentation et réassemblage des données - Le périphérique émetteur segmente les données d'application en blocs de taille appropriée. Le périphérique récepteur réassemble les segments en données d'application.

  • Identification des applications - Pour que les flux de données atteignent les applications auxquelles ils sont destinés, la couche transport doit identifier l'application cible. Pour cela, la couche transport affecte un identificateur à chaque application, appelé numéro de port.
