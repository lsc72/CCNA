Services de fichiers

Le protocole FTP (File Transfer Protocol) a été standardisé en 1971, afin de permettre les transferts de données entre un client et un serveur. Un client FTP est une application qui s'exécute sur un ordinateur. Il est utilisé pour placer ou extraire des données sur un serveur qui exécute le protocole FTP en tant que service.

Comme le montre l'illustration ci-contre, pour transférer les données, le protocole FTP nécessite deux connexions entre le client et le serveur : l'une pour les commandes et les réponses, l'autre pour le transfert de fichiers en lui-même.

Cependant, le protocole FTP présente de nombreuses faiblesses en matière de sécurité. C'est pourquoi il est nécessaire d'utiliser des services de transfert de fichiers plus sécurisés, notamment :

  • File Transfer Protocol Secure (FTPS) - Un client FTP peut demander que la session de transfert de fichiers soit chiffrée à l'aide de TLS. Le serveur de fichiers peut accepter ou refuser la requête.

  • SSH File Transfer Protocol (SFTP) - En tant qu'extension du protocole SSH (exécutant le protocole FTP en tant que service), SFTP peut être utilisé pour établir une session de transfert de fichiers.

  • Secure Copy (SCP) - SCP utilise également SSH pour sécuriser les transferts de fichiers.
