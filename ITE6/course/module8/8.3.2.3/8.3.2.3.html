Services Web

Les ressources Web sont fournies par un serveur Web. L'hôte accède aux ressources Web à l'aide du protocole HTTP ou HTTP sécurisé (HTTPS). HTTP correspond à un ensemble de règles permettant d'échanger du texte, des graphiques, des sons et des vidéos sur le Web. HTTPS ajoute des services de chiffrement et d'authentification à l'aide du protocole SSL (Secure Sockets Layer) ou du nouveau protocole TLS (Transport Layer Security). HTTP utilise généralement le port 80. HTTPS fonctionne sur le port 443.

Pour mieux comprendre l'interaction entre le navigateur Web et le serveur Web, voyons comment une page Web s'ouvre dans un navigateur. Dans cet exemple, utilisez l'URL http://www.cisco.com/index.html.

Dans un premier temps, le navigateur interprète les trois parties de l'URL : (voir illustration 1)

1. http (protocole ou schéma)

2. www.cisco.com (nom du serveur)

3. index.html (nom du fichier demandé)

Le navigateur fait ensuite appel à un serveur de noms pour convertir l'adresse www.cisco.com en une adresse numérique, qu'il utilise pour se connecter au serveur. Le navigateur envoie une requête GET au serveur et demande le fichier index.html en se conformant à la norme HTTP, voir l'illustration 2. Le serveur renvoie le code HTML de cette page Web au navigateur du client, voir l'illustration 3. Enfin, comme le montre l'illustration 4, le navigateur déchiffre le code HTML et met en forme la page pour la fenêtre du navigateur.
