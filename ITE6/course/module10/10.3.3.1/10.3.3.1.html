E-mails : introduction

La structure du système de messagerie repose sur des serveurs et des clients. Les serveurs de messagerie transmettent les e-mails envoyés par leurs utilisateurs. Les utilisateurs se servent des clients de messagerie pour composer, lire et consulter leurs e-mails. Il peut s'agir d'applications Web ou d'applications autonomes. Les clients de messagerie autonomes dépendent de chaque plate-forme. Cette section traite des clients de messagerie des appareils mobiles.

Les informations suivantes sont nécessaires pour configurer un compte de messagerie :

  • Nom complet : vous pouvez utiliser votre nom réel, votre pseudonyme ou un autre nom. Il s'agit du nom que verront les personnes que vous contactez.

  • Adresse e-mail : adresse à laquelle vous recevrez vos e-mails. Une adresse e-mail est constituée d'un nom d'utilisateur suivi du symbole @ et du domaine du serveur de messagerie (nom@exemple.net).

  • Protocole utilisé par le serveur de messagerie pour courrier entrant : les différents protocoles offrent des services de messagerie différents.

  • Nom des serveurs pour courrier entrant et courrier sortant : ces noms sont fournis par l'administrateur réseau ou le FAI.

  • Nom d'utilisateur : nom utilisé pour se connecter aux serveurs de messagerie.

  • Mot de passe du compte : ce mot de passe doit être complexe, car les comptes de messagerie sont souvent utilisés sur différents sites Web.

Les protocoles de messagerie utilisés comprennent :

  • Protocole POP3 (Post Office Protocol version 3)

  • IMAP (Internet Message Access Protocol)

  • SMTP (Simple Mail Transfer Protocol)

  • MIME (Multipurpose Internet Mail Extensions)

  • SSL (Secure Socket Layer)

Vous devez savoir comment configurer un appareil pour qu'il accepte le format correct de courrier entrant. Vous pouvez configurer le logiciel client de messagerie à l'aide d'un assistant.

POP3

Le protocole Post Office Protocol 3 (POP3) récupère les e-mails auprès d'un serveur distant, via TCP/IP. POP3 ne laisse aucune copie des e-mails sur le serveur. Toutefois, certaines implémentations permettent aux utilisateurs de conserver les e-mails pendant un certain temps. Le protocole POP3 prend en charge les utilisateurs disposant d'une connexion intermittente, par exemple par ligne commutée. Avec le protocole POP3, l'utilisateur peut se connecter, télécharger les e-mails à partir du serveur, puis se déconnecter. Le protocole POP3 utilise généralement le port 110.

IMAP

Le protocole Internet Mail Access Protocol (IMAP) permet aux clients de messagerie locaux de récupérer les e-mails sur un serveur. Tout comme le protocole POP3, l'IMAP vous permet de télécharger les e-mails à partir d'un serveur de messagerie à l'aide d'un client de messagerie. La différence est que le protocole IMAP permet à l'utilisateur d'organiser ses e-mails sur le serveur de messagerie du réseau, ainsi que d'en télécharger des copies. Les originaux sont conservés sur le serveur de messagerie du réseau. Contrairement au protocole POP3, le protocole IMAP laisse généralement les e-mails originaux sur le serveur jusqu'à ce que l'utilisateur les mette dans un dossier personnel au sein de l'application de messagerie. Le protocole IMAP synchronise les dossiers d'e-mails entre le serveur et le client. IMAP est plus rapide que le protocole POP3, mais nécessite plus d'espace disque sur le serveur et plus de ressources processeur. La version la plus récente du protocole IMAP est IMAP4. Le protocole IMAP4 est souvent utilisé dans les réseaux de grande taille, par exemple sur les campus. Le protocole IMAP utilise généralement le port 143.

SMTP

Le protocole SMTP (Simple Mail Transfer Protocol) est un protocole texte qui transmet les messages sur un réseau TCP/IP. Il s'agit d'un format de messagerie texte utilisant uniquement le codage ASCII. Le protocole SMTP doit être implémenté pour envoyer des e-mails. Les e-mails sont envoyés depuis un client de messagerie vers un serveur de messagerie, ou d'un serveur de messagerie à un autre. Il est à noter que les e-mails sont envoyés uniquement lorsque les destinataires sont identifiés et vérifiés. Ce protocole utilise généralement le port 25.

MIME

Le protocole Multipurpose Internet Mail Extensions (MIME) étend le format des e-mails afin d'inclure le texte ASCII, ainsi que d'autres formats tels que des images et des documents de traitement de texte. La norme MIME est généralement utilisée conjointement avec le protocole SMTP.

SSL

Le protocole SSL (Secure Socket Layer) a été développé pour transmettre des fichiers en toute sécurité. Toutes les données échangées entre le client de messagerie et le serveur de messagerie sont en effet chiffrées. Pour configurer un client de messagerie devant utiliser le protocole SSL, il est nécessaire de sélectionner le bon port du serveur de messagerie.

Exchange

Exchange est un logiciel de serveur de messagerie, de gestion des contacts et de calendrier créé par Microsoft. Il utilise une interface de messagerie propriétaire appelée MAPI (Messaging Application Programming Interface). L'interface MAPI est utilisée par Microsoft Office Outlook pour se connecter aux serveurs Exchange et exploiter les services de messagerie et de gestion des calendriers et des contacts.
