Applications et sources de contenu

Les applications, appelées « apps », sont les programmes exécutés sur les appareils mobiles. Sur ces derniers, différentes applications de base sont préinstallées. Elles servent notamment à téléphoner, à envoyer et recevoir des e-mails, à écouter de la musique, à prendre des photos, à lire des vidéos ou à jouer à des jeux vidéo. D'autres permettent d'accéder à des informations, à personnaliser l'appareil et à améliorer la productivité.

Les applications des appareils mobiles sont l'équivalent des programmes des ordinateurs. Cependant, au lieu d'être installées à partir d'un disque optique, elles sont téléchargées. Certaines applications sont gratuites alors que d'autres sont payantes. De nombreuses sources de contenu sont disponibles sur les appareils mobiles Android :

  • Google Play (voir l'illustration 1)

  • Amazon App Store

  • Androidzoom

  • AppsAPK

  • 1Mobile

En outre, on peut trouver des applications Android sur de nombreux autres sites Web. Il est donc important d'installer uniquement les applications provenant de sources de confiance. Si un site suspect vous propose l'application que vous recherchez, vérifiez d'abord si vous pouvez plutôt la télécharger sur Google Play ou sur Amazon App Store. Les applications téléchargées à partir de sources sécurisées sont moins susceptibles de contenir du code malveillant. Google Play restreint l'installation des applications aux appareils compatibles.

Vous trouverez sur certains sites des codes QR, voir l'illustration 2. Ces codes sont similaires aux codes-barres, mais ils contiennent beaucoup plus de données. Pour lire un code QR, une application spécifique doit accéder à l'appareil photo de l'appareil mobile pour lire le code. Ce dernier contient un lien vers un site Web qui permet de télécharger directement une application. Soyez prudent lorsque vous utilisez un code QR et n'acceptez un téléchargement ou une installation que si vous êtes redirigé vers une source de confiance.

Sur Android, les applications sont distribuées dans un format appelé APK, ou Android Application Package. Lorsque l'application Android est compilée et prête à être diffusée, elle est placée dans un fichier APK. En plus du code compilé de l'application, l'APK contient les ressources et les certificats nécessaires au bon fonctionnement de l'application.

L'App Store d'Apple (voir l'illustration 3) est la seule source de contenu disponible pour les utilisateurs d'iOS. Ces derniers ont ainsi l'assurance que le contenu téléchargeable ne contient pas de programmes malveillants, que les applications respectent des règles strictes en matière de performances et que les droits d'auteur sont respectés.

À l'instar de l'App Store d'Apple ou de Google Play, Microsoft a lancé son propre magasin d'applications en 2012. Depuis Windows 8 et Windows Server 2012, le Windows Store, voir l'illustration 4, offre aux utilisateurs de Windows la possibilité de rechercher, de télécharger et d'installer des applications Windows Store (également appelée applications Metro).

D'autres types de contenu peuvent être téléchargés sur des appareils mobiles. Tout comme pour les applications, certains contenus sont gratuits alors que d'autres sont payants. Le contenu que vous possédez peut également être transféré sur des appareils mobiles à l'aide d'un câble de données ou d'une connexion Wi-Fi. Voici des exemples de contenu :

  • Musique

  • Émissions de télévision

  • Films

  • Magazines

  • Livres

Méthode pull ou push

Deux méthodes principales sont utilisées pour installer les applications et le contenu sur les appareils mobiles : push et pull. Lorsqu'un utilisateur lance l'application Google Play ou App Store sur un appareil mobile, les applications et le contenu téléchargés sont extraits d'un serveur et envoyés vers l'appareil (pull).

Avec Android, l'utilisateur peut parcourir Google Play sur un ordinateur portable ou de bureau et acheter du contenu. Ce contenu est alors « poussé » (push) sur l'appareil Android par le serveur. De même, les utilisateurs d'iOS peuvent acheter du contenu sur iTunes sur un ordinateur de bureau ou portable. Ce contenu sera ensuite envoyé (push) sur l'appareil iOS.

Lorsqu'une application Android est installée, la liste de toutes les autorisations requises s'affiche, voir l'illustration 5. Vous devez accepter de donner ces autorisations à une application pour pouvoir l'installer. Lisez toujours attentivement la liste des autorisations requises et n'installez pas d'applications nécessitant un accès à des éléments et des fonctionnalités dont elles ne devraient pas avoir besoin.

Sideloading

Il existe une autre méthode pour installer des applications sur les appareils mobiles Android. Les applications peuvent être téléchargées à partir de différentes sources sur Internet, puis transférées vers un appareil mobile via le Wi-Fi, une connexion Bluetooth, un câble de données, etc. C'est ce qu'on appelle le « sideloading ». Cependant, le sideloading des applications n'est pas recommandé, car de nombreuses sources d'applications ne sont pas fiables. N'installez que les applications distribuées par des sources et des développeurs de confiance.

Remarque : les appareils iOS ne sont pas compatibles par défaut avec le sideloading.
