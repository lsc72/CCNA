Outils de programmation

La section Outils d'administration comporte également des outils de programmation. Ces outils sont destinés à faciliter la programmation de certaines applications.

Services de composants

Développés par Microsoft, de nombreux outils permettent d'activer le partage des objets de données. COM, COM+ et DCOM sont des exemples de ces outils. Un objet de données est tout simplement un élément de programme. L'outil Services de composants, voir l'illustration 1, permet d'apporter des modifications à un programme, pour qu'il fonctionne mieux ou différemment.

Pour accéder à l'outil Services de composants sous Windows 8 et Windows 7, sélectionnez :

Panneau de configuration > Outils d'administration > Services de composants

Pour accéder à l'outil Services de composants Sous Windows Vista, sélectionnez :

Rechercher > dcomcnfg

Source de données (ODBC)

ODBC (Open Database Connectivity), voir l'illustration 2, est une interface de programmation conçue pour autoriser l'accès aux systèmes de gestion de bases de données. ODBC a été créé pour que l'accès à ces systèmes ne dépende ni d'un système de base de données, ni d'un système d'exploitation. Par exemple, cet outil de programmation s'avère utile pour la création d'une base de données partagée.

Pour accéder à Source de données (ODBC) sous Windows 7 ou Windows Vista, sélectionnez :

Panneau de configuration > Outils d'administration > Sources de données (ODBC)

Pour accéder aux sources de données (ODBC) sous Windows 8, sélectionnez :

Panneau de configuration > Outils d'administration > Source de données ODBC
