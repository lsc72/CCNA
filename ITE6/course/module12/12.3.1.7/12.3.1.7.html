Gérer les utilisateurs

Les tâches de maintenance régulières que doivent effectuer les administrateurs consistent à créer et à supprimer les utilisateurs sur le réseau, changer les mots de passe des comptes ou modifier les autorisations des utilisateurs.

Les opérations suivantes peuvent être effectuées lors de la gestion des comptes d'utilisateurs locaux :

  • Créer un compte d'utilisateur local

  • Réinitialiser le mot de passe d'un compte d'utilisateur local

  • Désactiver ou activer un compte d'utilisateur local

  • Supprimer un compte d'utilisateur local

  • Renommer un compte d'utilisateur local

  • Attribuer un script d'ouverture de session à un compte d'utilisateur local

  • Attribuer un dossier de base à un compte d'utilisateur local

Deux outils peuvent être utilisés pour effectuer ces opérations :

  • Contrôle de compte d'utilisateur (UAC) : pour ajouter, supprimer ou modifier les attributs des utilisateurs. Une fois connecté en tant qu'administrateur, utilisez le contrôle de compte d'utilisateur pour configurer les paramètres afin d'empêcher tout code malveillant d'obtenir les privilèges d'administrateur.

  • Gestionnaire Utilisateurs et groupes locaux : peut être utilisé pour créer et gérer les utilisateurs et les groupes qui sont stockés localement sur un ordinateur.

Remarque : vous devez disposer de privilèges administrateur pour pouvoir gérer les utilisateurs.

Pour ouvrir le Contrôle de compte d'utilisateur, procédez comme suit :

Panneau de configuration > Comptes utilisateurs > Gérer un autre compte

Pour ouvrir le gestionnaire Utilisateurs et groupes locaux, procédez comme suit :

Panneau de configuration > Outils d'administration > Gestion de l'ordinateur > Utilisateurs et groupes locaux

Avec les utilisateurs et les groupes locaux, vous pouvez limiter la capacité des utilisateurs et des groupes à effectuer certaines opérations en leur attribuant des droits et des autorisations. Le droit permet à l'utilisateur d'effectuer certaines opérations sur un ordinateur, par exemple la sauvegarde de fichiers et de dossiers ou l'arrêt de l'ordinateur. Une autorisation est une règle associée à un objet (en général un fichier, un dossier ou une imprimante) et qui définit quels utilisateurs ont accès à un objet et de quelle manière.

Dans la fenêtre Utilisateurs et groupes locaux, double-cliquez sur Utilisateurs. Il y a deux comptes prédéfinis :

  • Administrateur : par défaut, ce compte est désactivé. Lorsqu'il est activé, ce compte a le contrôle total de l'ordinateur et il peut attribuer des droits d'utilisateur et des autorisations de contrôle d'accès aux utilisateurs en fonction de leurs besoins. Le compte Administrateur est membre du groupe Administrateurs de l'ordinateur. Le compte Administrateur ne peut jamais être supprimé ni retiré du groupe Administrateurs, mais il peut être renommé ou désactivé.

  • Invité : par défaut, ce compte est désactivé. Il sert aux utilisateurs qui n'ont pas de compte sur l'ordinateur. Par défaut, ce compte ne nécessite pas de mot de passe. Il est membre du groupe Invités par défaut, ce qui permet à l'utilisateur d'ouvrir une session sur l'ordinateur.

Pour ajouter un utilisateur, cliquez sur le menu Action et sélectionnez Nouvel utilisateur. Cette commande ouvre la fenêtre Nouvel utilisateur, voir l'illustration 2. Vous pouvez attribuer un nom d'utilisateur, un nom complet, une description et définir les options du compte.

Double-cliquez sur un utilisateur ou cliquez avec le bouton droit et choisissez Propriétés pour ouvrir la fenêtre des propriétés de l'utilisateur, voir l'illustration 3. Cette fenêtre vous permet de modifier les options utilisateur définies lorsque l'utilisateur a été créé. Elle permet aussi de verrouiller un compte. La fenêtre vous permet également d'affecter un utilisateur à un groupe à l'aide de l'onglet Membre ou de contrôler les dossiers auxquels l'utilisateur a accès à l'aide de l'onglet Profil.

Remarque : il existe également un autre type important de compte appelé Utilisateur avancé. Ce compte possède la plupart des droits de l'administrateur, notamment celui d'installer des programmes ou de modifier des paramètres du pare-feu, mais il ne dispose pas de tous les privilèges de l'administrateur, pour des raisons de sécurité.
