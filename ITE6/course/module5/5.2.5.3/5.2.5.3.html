Extensions et attributs de fichiers

Les fichiers d'une structure de répertoires respectent une convention d'appellation Windows :

  • Un maximum de 255 caractères est autorisé.

  • Les caractères tels que la barre oblique et la barre oblique inverse (/ \) ne sont pas acceptés.

  • Une extension de trois ou quatre lettres est ajoutée au nom du fichier pour identifier le type de fichier.

  • Les noms de fichiers ne sont pas sensibles à la casse.

Par défaut, les extensions de fichiers sont cachées. Pour les afficher, vous devez désactiver le paramètre Masquer les extensions des fichiers dont le type est connu dans l'utilitaire Options des dossiers du Panneau de configuration, comme le montre l'illustration ci-contre.

Pour afficher les extensions des fichiers, sélectionnez :

Panneau de configuration > Options des dossiers > Affichage et désactivez la case à cocher Masquer les extensions des fichiers dont le type est connu.

Les extensions de fichiers suivantes sont couramment utilisées :

  • .docx : Microsoft Word (version 2007 et ultérieures)

  • .txt : texte ASCII uniquement

  • .jpg : format d'image

  • .pptx : Microsoft PowerPoint

  • .zip : format de compression

La structure de répertoires gère un jeu d'attributs pour chaque fichier, qui contrôle la manière dont le fichier peut être affiché ou modifié. Les attributs de fichier les plus répandus sont les suivants :

  • R : le fichier est en lecture seule.

  • A : le fichier sera archivé à la prochaine sauvegarde du disque.

  • S : le fichier est marqué comme étant un fichier système et un avertissement est donné en cas de tentative de suppression ou de modification.

  • H : le fichier est caché dans l'affichage du contenu des répertoires.
